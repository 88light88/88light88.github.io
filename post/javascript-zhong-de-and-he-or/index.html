JavaScript 中的 && 和 ||

2020-06-24 JavaScript

今天在阅读Underscore.js源代码时看到这以下这样一段代码：

var root = typeof self == 'object' && self.self === self && self ||
            typeof global == 'object' && global.global === global && global ||
            Function('return this')() ||
            {};

在我们以前的学习和使用中，逻辑表达式最终会返回来一个布尔值，而这段代码最终会返回一个对象给root变量。这是怎么回事，它是怎么返回的，它是什么样的原理就不是很清楚了。后来通过在google查询资料才明白该使用方式，下面就来记录一下。

&& 和 ||

1、&&（与）：表示并且的关系。

情况一：&& 两边是条件或者true和false，此情况flag的将会接收到一个布尔值。

    var flag = 1==1 && 2==2 ;

情况二：

    var flag = 0 && 1 && 2 ;

情况三：

    var flag = 1==1 && 2 ；

以上的代码就类似于文章开头中的Underscore.js源码中的那一小段代码，我刚开始是这样理解这段代码的，在javascript中数值类型的值0或NaN为false，其他为true。在上面的例子中使用的是数值，所以我理解成 false && true && true 结果为 false ，可是在实践了之后大错特错，并不会这样返回这样的结果而是返回了0。后来在查询资料以及实验中得出一下的结论：
（1）如果逻辑运算符两侧都是布尔值（可以返回布尔值的条件），则在运算完成之后返回布尔值true或false。
（2）如果逻辑运算符两边为非布尔值它会去从左到右去将数据转换为布尔，依次判断是否存在false，如果存在则返回当前false位置的那个值，在上面情况二的例子就表示返回的是 0。如果依次转换并检测到结尾后没有为false的值，就返回最后一个值。
（3）逻辑运算符两侧既包含条件也包含值，类似于情况三这样。依次向右判断条件和值（转换）都为真，则返回最后一个值，否则就返回false

总结：优先返回第一个为假的值，否则返回true或最后一个值。
